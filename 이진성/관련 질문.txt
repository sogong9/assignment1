3) 교수님 안녕하세요. Use case description 관련 질문드립니다. 
extend하는 use case description은 하나의 UI 화면 기준으로 해당 main use case에 포함되어 기술하면 된다고 하셨던 부분과 extension을 step by step으로 쪼개서 설명해야 한다는 점을 고려했을 때, main use case와 extension 등장 시점(예를 들어 after step 2)을 표기한 후, 또 다른 use case 를 만들어 extension의 description을 적는지, 아니면 하나의 main use case 에 extension의 등장 시점와 extension의 step by step을 함께 표기하면 되는지 여쭤보고 싶습니다.

감사합니다.			48


--  > extend하는 use case가 간단하면 main use case와 같이 하나의 description에 기술합니다.
 	다만, extend하는 use case가 많아지면 오히려 더 복잡하고 이해하기 힘들 수 있기 때문에 분리해서 기술해도 됩니다.
 	둘 다 해당 기능을 포함하기 때문에 결과는 동일합니다. 다만, 분석하고 이해하기 쉬운 방향으로 결정하면 됩니다.
